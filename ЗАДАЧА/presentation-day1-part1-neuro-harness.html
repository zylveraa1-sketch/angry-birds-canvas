opencode: лимиты и почему это рабочий выбор

Что важно запомнить

  • 200 запросов на 5 часов для Big Pickle и других бесплатных моделей.
  • Лимиты считаются в 5-часовом окне.
  • Разные модели/провайдеры дают разный расход лимита.

Правило на практике

  • Если застряли, уменьшайте задачу.
  • Сначала MVP, потом улучшения.
  • Держите ритм: задача → проверка → commit.

Что показываем

  • Как дать чёткую задачу на новый файл.
  • Как сразу указать путь и цель файла.
  • Как проверить, что файл реально создан.
Создай файл `src/game/config.js` с базовыми настройками canvas.

Plan mode vs Build mode

Plan mode

  • Режим планирования и исследования.
  • Tools для управления компьютером запрещены.
  • Системный промпт заточен под анализ, а не под выполнение.

Build mode

  • Обычный рабочий режим для реальной разработки.
  • Можно выполнять действия через tools и вносить изменения.
  • Хороший тон: сначала полный план, потом реализация по плану.
Рекомендация: Plan mode -> утвердили план -> Build mode -> реализация и проверки.

Оптимальные форматы данных для ИИ

TXT

  • Обычный текст: минимум структуры.
  • Для сложных задач читается хуже.
сделать игру про мяч добавить счет таймер паузу если проигрыш то экран поражения нужны ассеты и звуки

MD (Markdown)

  • Универсален: блоки, списки, заголовки.
  • Можно выделять важное: **текст**.
# MVP игры ## Цель Сделать 2D-игру за 3 дня. ## Фичи - движение - счёт - экран поражения **Критерий готовности:** игра запускается локально.

JSON

  • Идеален для конфигов и настроек.
  • Структурированные данные для tools/API.
{ "game": { "title": "Ball Rush", "canvas": { "width": 1280, "height": 720 }, "player": { "speed": 7, "lives": 3 } } }

Установка opencode (скрин интерфейса)

Linux

  • Используем CLI-утилиту.
  • Ставим по инструкции курса.
  • Проверка: `opencode --help`.

Windows

  • Доступны CLI и GUI.
  • Проверяем запуск на своём проекте.
  • Если нет доступа/лимита в других сервисах — остаёмся в opencode.

Cursor: дополнительный инструмент

Когда используете

  • Только как запасной GUI-вариант.
  • Проверяете лимит до начала работы.
  • При лимите 0 сразу возвращаетесь в opencode.

Ограничение

  • Маленький общий лимит на месяц.
  • В аудитории у новых аккаунтов может сработать обнуление.
  • Поэтому критически важные задачи делаем не здесь.
